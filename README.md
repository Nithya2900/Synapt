# Agentic RAG — Indian IT Company Financials

A small LLM agent that answers questions over a mixed corpus:  
annual reports (unstructured) + financial CSV (structured) + live web search.

---

## What It Does

Ask a question. The agent picks the right tool (or tools), retrieves grounded information,
composes a cited answer, and logs every step. It refuses investment advice and stops
gracefully after a hard cap of 8 tool calls.

**Examples:**
```
"What was TCS operating margin in FY2024?"         → query_data
"What reason did Infosys give for margin decline?" → search_docs
"What is Wipro's current stock price?"             → web_search
"How did TCS and Infosys margins compare, and why?" → query_data + search_docs
"Should I invest in TCS?"                          → refusal, no tool called
```

## Setup (3 steps)

### 1. Clone and install
```bash
git clone <your-repo-url>
cd agentic-rag
pip install -r requirements.txt
```

### 2. Add API keys
```bash
cp .env.example .env
# Edit .env and add:
#   ANTHROPIC_API_KEY=your_key_here
#   TAVILY_API_KEY=your_key_here  (optional — mock results used without it)
```

Get your gemini key at: https://aistudio.google.com
Get your Tavily key at: https://tavily.com (free tier, 1000 searches/month)

### 3. Build index and database
```bash
python setup.py
```
This downloads the sentence-transformers model (~90MB, one time) and builds the FAISS index and SQLite DB.

---

## Running the Agent

**Single question:**
```bash
python agent.py "What was Infosys operating margin in FY2024?"
```

**Interactive mode:**
```bash
python agent.py
```
Type `save` after any answer to save the trace as JSON.

**Run full evaluation:**
```bash
python evaluate.py
```
Produces `EVALUATION.md` with results for all 20 questions.

## Known Failure Modes

1. **Ambiguous NL-to-SQL**: Rule-based SQL generation fails on complex phrasings like "best performing company over time". Fix: use LLM-generated SQL.

2. **Multi-tool ordering**: The agent occasionally calls web_search before exhausting structured sources for hybrid questions. Fix: add a planning step before the loop.

3. **Mock web search**: Without a Tavily API key, web results are static mock data. Real current prices and news require the key.

4. **Text-only annual reports**: The corpus uses pre-converted text files, not real PDFs. Page numbers in citations are not available without PDF extraction.
